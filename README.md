# ชุด Agents กฎหมายไทยสำหรับ OpenCode

โปรเจกต์นี้เป็นชุด OpenCode agents แบบ project-scoped สำหรับงานกฎหมายไทย งานค้นคว้ากฎหมาย งานตรวจสัญญา งานวางแผนข้อพิพาท งาน compliance งานแปลเอกสารกฎหมาย และงานร่างหนังสือราชการไทย

## รายชื่อ Agents

- `thai-legal-context-manager`: agent หลัก ทำหน้าที่จัดการบริบท แยกงาน ส่งงานให้ subagents และรวมคำตอบกลับให้ผู้ใช้
- `thai-lawyer`: วิเคราะห์ข้อกฎหมายไทย ประเด็นข้อพิพาท ความเสี่ยง และข้อจำกัดทางกฎหมาย
- `thai-official-writer`: ร่างหนังสือราชการ บันทึกข้อความ คำสั่ง ประกาศ รายงานการประชุม และสำนวนทางราชการไทย
- `thai-legal-researcher`: ค้นคว้าและตรวจสอบแหล่งกฎหมายไทยที่เป็นทางการ citation แบบฟอร์ม กำหนดเวลา ค่าธรรมเนียม และข้อมูลปัจจุบัน
- `thai-contract-reviewer`: ตรวจสัญญาไทยหรือสัญญาสองภาษา วิเคราะห์ clause ความเสี่ยง ภาระหน้าที่ และประเด็นเจรจา
- `thai-litigation-planner`: วางแผนข้อพิพาท คดี การร้องเรียน การอุทธรณ์ พยานหลักฐาน ขั้นตอน และทางเลือกในการระงับข้อพิพาท
- `thai-compliance-officer`: วิเคราะห์ compliance ตามกฎหมายไทย เช่น PDPA ใบอนุญาต การรายงาน หน้าที่กำกับดูแล และ internal controls
- `thai-legal-translator`: แปลเอกสารกฎหมายและเอกสารราชการไทย-อังกฤษ โดยรักษาความหมายทางกฎหมายและรูปแบบเอกสาร

## วิธีใช้งาน

เปิด directory นี้ด้วย OpenCode แล้วเลือกหรือเรียกใช้ agent หลัก:

```text
@thai-legal-context-manager
```

โดยปกติควรเริ่มจาก `thai-legal-context-manager` เพราะ agent นี้จะเลือก subagent ที่เหมาะสมให้เอง และส่งงานระหว่าง agents ด้วย JSON protocol ภายใน

## ตัวอย่าง Prompt

วิเคราะห์กฎหมายแรงงาน:

```text
ช่วยวิเคราะห์ข้อกฎหมายไทยกรณีลูกจ้างถูกเลิกจ้างโดยไม่ได้รับค่าชดเชย
```

ร่างหนังสือราชการ:

```text
ร่างหนังสือราชการถึงนายอำเภอ ขอความอนุเคราะห์ตรวจสอบข้อมูลที่ดิน
```

ตรวจร่างเอกสาร:

```text
ตรวจร่างหนังสือนี้ทั้งในแง่ข้อกฎหมายและสำนวนราชการไทย: ...
```

ตรวจสัญญา:

```text
ตรวจสัญญาบริการฉบับนี้ ระบุความเสี่ยงและข้อเสนอแก้ไข: ...
```

วางแผนข้อพิพาท:

```text
ช่วยวางแผนข้อพิพาทและหลักฐานที่ต้องเตรียมก่อนฟ้องคดีแพ่ง
```

ตรวจ compliance:

```text
ตรวจ compliance PDPA สำหรับระบบเก็บข้อมูลลูกค้า
```

แปลเอกสาร:

```text
แปลหนังสือราชการฉบับนี้เป็นภาษาอังกฤษโดยคงรูปแบบราชการไทย
```

## Communication Protocol

`thai-legal-context-manager` ส่งงานให้ subagents ด้วย JSON protocol ชื่อ `thai-legal-agents.v1`

subagents จะตอบกลับเป็น JSON ที่มีข้อมูลสำคัญ เช่น:

- `findings`: ข้อค้นพบ ข้อกฎหมาย ความเสี่ยง หรือข้อเสนอแนะ
- `questions`: คำถามที่ต้องขอข้อมูลเพิ่ม
- `draft`: ร่างเอกสารหรือข้อความ
- `citations`: แหล่งอ้างอิงหรือเอกสารทางการ
- `warnings`: ข้อควรระวังหรือข้อจำกัด

จากนั้น context manager จะสรุปและจัดรูปคำตอบเป็นภาษาไทยสำหรับผู้ใช้ โดยไม่แสดง JSON ภายใน เว้นแต่ผู้ใช้ต้องการดู

รายละเอียด protocol อยู่ในไฟล์ `.opencode/agents/thai-legal-context-manager.md`

## ข้อควรระวัง

Agents ชุดนี้ช่วยให้ข้อมูลกฎหมาย วิเคราะห์ความเสี่ยง ค้นคว้าแหล่งอ้างอิง และช่วยร่างเอกสาร แต่ไม่ใช่ความเห็นทางกฎหมายฉบับสุดท้ายจากทนายความผู้ได้รับใบอนุญาต

ก่อนนำไปยื่น ใช้อ้างอิงอย่างเป็นทางการ หรือใช้ตัดสินใจในเรื่องสำคัญ ควรตรวจสอบกฎหมาย กฎ ระเบียบ ประกาศ แบบฟอร์ม กำหนดเวลา ค่าธรรมเนียม และแนวปฏิบัติของหน่วยงานที่เป็นปัจจุบันอีกครั้ง
